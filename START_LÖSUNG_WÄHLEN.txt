DEINE BUILD-FEHLER LÖSUNGS-TOOLS:
════════════════════════════════════════════════════════════════════════════════

1️⃣  BUILD_FEHLER_SCHNELLE_LÖSUNG.txt
    👉 STARTE HIER!
    └─ Schneller Überblick über alle Probleme
    └─ 3 verschiedene Lösungs-Optionen
    └─ Manuelle Anleitung mit Bildern

2️⃣  RemoveDuplicates.ps1  ⭐ EMPFOHLEN
    👉 Automatisches PowerShell-Script
    └─ Findet alle Duplikate automatisch
    └─ Löscht Compile-Output (bin/obj)
    └─ Zeigt Lösungs-Schritte an
    
    SO NUTZT DU ES:
    ───────────────────────────────────────
    1. Öffne PowerShell als Administrator
    2. Navigiere zum Projekt-Ordner:
       cd C:\...\MaterialManager_R03
    3. Führe aus:
       .\RemoveDuplicates.ps1
    4. Folge den Anweisungen

3️⃣  FIX_PROJEKT_SCHNELL.bat
    👉 Semi-Automatischer Fixer
    └─ Beendet Visual Studio & Prozesse
    └─ Löscht bin/obj
    └─ Öffnet Visual Studio neu
    
    SO NUTZT DU ES:
    ───────────────────────────────────────
    1. Doppelklick auf: FIX_PROJEKT_SCHNELL.bat
    2. Warte bis Visual Studio öffnet
    3. Folge den Anweisungen im Fenster

4️⃣  FEHLER_FIX_ANLEITUNG.txt
    👉 Manuelle Schritt-für-Schritt Anleitung
    └─ Detaillierte Erklärungen
    └─ Visual Studio Befehle
    └─ Wenn automatische Lösungen nicht funktionieren

5️⃣  CLEANUP_DUPLIKATE.bat
    👉 Erstellt Backups
    └─ Sichert alte Dateien in BACKUP_BEFORE_CLEANUP/
    └─ Räumt auf

WAS IST DAS PROBLEM?
════════════════════════════════════════════════════════════════════════════════

❌ FEHLER: "Der Typ XYZ enthält bereits eine Definition für ABC"

GRUND:
• Code wurde doppelt eingefügt
• Alte Definition wurde nicht gelöscht
• Beide Versionen existieren noch

BEISPIEL:
❌ Falsch - Code existiert 2x:
   public ObservableCollection<MaterialItem> Materialien { get; set; }
   // ... viel Code ...
   public ObservableCollection<MaterialItem> Materialien { get; set; }  ← Duplikat!

✅ Richtig - Code existiert nur 1x:
   public ObservableCollection<MaterialItem> Materialien { get; set; }

BACKUP-SICHERHEIT:
════════════════════════════════════════════════════════════════════════════════

Falls etwas schiefgeht, gibt es Backups in:
📁 BACKUP_BEFORE_CLEANUP/

Darin sind die Dateien vom Anfang des Cleanup-Prozesses gespeichert.

So stellst du wieder her:
1. Öffne BACKUP_BEFORE_CLEANUP/
2. Kopiere .bak Dateien
3. Einfügen in Projekt-Root
4. Benenne um: entferne ".bak"
5. Visual Studio reload
